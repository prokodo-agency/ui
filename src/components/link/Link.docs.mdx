# Link

Themed anchor/navigation link component. Wraps Next.js `Link` with design-system typography, color variants, and island architecture support for client-side navigation.

## When to use

Use `Link` for inline text links within prose, navigation lists, and CTAs where a button is semantically incorrect. For primary actions use `Button`.

## Accessibility (WCAG 2.2)

- Renders a native `<a>` element â€” fully keyboard and screen-reader accessible
- `aria-label` overrides the visible text for screen readers when needed
- `rel="noopener noreferrer"` is applied automatically for external links
- Meaningful link text: describe the destination, not the action ("View pricing" not "Click here")

## âś… Do

- Use descriptive link text that makes sense out of context (screen reader list view)
- Set `external` or `target="_blank"` for links that open in a new tab and inform users
- Use `color` to align the link color with its surrounding context

## âťŚ Don't

- Don't use `Link` to trigger JavaScript actions â€” use `Button` with `variant="text"`
- Don't use vague text like "here", "click here", or "read more" without surrounding context
- Don't open links in a new tab without warning

## Design Tokens

Customise `Link` via CSS custom properties on `:root` or a scoped ancestor.

| Token                           | Default                                                   | Description                                            |
| ------------------------------- | --------------------------------------------------------- | ------------------------------------------------------ |
| `--pk-link-color`               | `color-mix(in srgb, var(--pk-color-fg) 55%, transparent)` | Default link text color                                |
| `--pk-link-color-hover`         | `var(--pk-color-fg)`                                      | Link color on hover/focus when no color variant is set |
| `--pk-link-color-primary`       | `var(--pk-color-brand)`                                   | Color when `color="primary"`                           |
| `--pk-link-color-secondary`     | `var(--pk-color-accent)`                                  | Color when `color="secondary"`                         |
| `--pk-link-color-info`          | `color-mix(in srgb, var(--pk-color-fg) 35%, transparent)` | Color when `color="info"`                              |
| `--pk-link-color-success`       | `var(--pk-color-success)`                                 | Color when `color="success"`                           |
| `--pk-link-color-warning`       | `var(--pk-color-warning)`                                 | Color when `color="warning"`                           |
| `--pk-link-color-error`         | `var(--pk-color-error)`                                   | Color when `color="error"`                             |
| `--pk-link-color-white`         | `var(--pk-palette-white)`                                 | Color when `color="white"`                             |
| `--pk-link-color-variant-hover` | `color-mix(in srgb, var(--pk-color-fg) 55%, transparent)` | Hover/focus color applied to colored variant links     |
