# Slider

A draggable range input for selecting a numeric value within a defined min/max range. Supports labelled tick marks, discrete fixed values, and custom step sizes.

## When to use

Use `Slider` for continuous or stepped numeric input where direct keyboard entry is impractical — budget filters, volume controls, date-range selectors. For precise numeric input use a number `Input`.

## Accessibility (WCAG 2.2)

- The thumb is a native `<input type="range">` with `aria-label`, `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`
- Tick labels are linked via `<datalist>` so screen readers can announce them
- Current value is announced on change via `aria-valuetext` with the formatted value and unit

### Keyboard behavior

- **Tab**: focus the slider thumb
- **← / → Arrow keys**: decrease / increase the value by one step
- **Home**: jump to minimum
- **End**: jump to maximum

## Design Tokens

Customise via CSS custom properties on `:root` or a scoped ancestor element.

| Token                       | Default                      | Description         |
| --------------------------- | ---------------------------- | ------------------- |
| `--pk-slider-track-height`  | `4px`                        | Track height        |
| `--pk-slider-track-bg`      | `var(--pk-color-border)`     | Unfilled track      |
| `--pk-slider-fill-from`     | `var(--pk-color-brand)`      | Fill gradient start |
| `--pk-slider-fill-to`       | `var(--pk-color-accent)`     | Fill gradient end   |
| `--pk-slider-thumb-size`    | `24px`                       | Thumb size          |
| `--pk-slider-mark-bg`       | `var(--pk-palette-grey-400)` | Mark dot colour     |
| `--pk-slider-mark-label-fg` | `var(--pk-input-helper-fg)`  | Mark label colour   |
| `--pk-slider-tooltip-bg`    | `var(--pk-color-fg)`         | Tooltip background  |

## ✅ Do

- Always display the current value numerically alongside the thumb
- Use `fixedValues` to constrain selection to meaningful breakpoints (e.g. pricing tiers)

## ❌ Don't

- Don't use `Slider` as the sole input for fine numeric precision — pair it with a read-out or allow direct input
- Don't rely on colour alone to indicate the filled track portion
