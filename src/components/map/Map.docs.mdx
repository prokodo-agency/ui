# Map

Renders an interactive Google Map with optional markers. The map reacts to the site’s `data-theme` attribute to automatically switch between a light and dark map style.

## When to use

Use `Map` when you need to display a geographic location or route. For decorative map graphics use `Image` instead.

## Styling

The map height is controlled via the `--pk-map-height` CSS custom property (default `500px`). Override it on a parent element:

```css
.my-contact-section {
  --pk-map-height: 400px;
}
```

## Accessibility (WCAG 2.2)

- Always provide a text alternative for the map location (address, description) near the component
- Interactive map controls must be keyboard accessible
- Color contrast of map overlays and annotations must meet 3:1 ratio (WCAG 1.4.11)
- Provide a fallback link to an external map service for users who cannot interact with the embedded map

## ✅ Do

- Include a plain-text address alongside the map for users who cannot view it
- Lazy-load the map to avoid blocking page render

## ❌ Don't

- Don’t rely solely on the map to communicate location — always include a text address
- Don’t embed maps in modals without explicit scroll-lock handling
