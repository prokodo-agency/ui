# Quote

A styled blockquote for pull quotes, testimonials, and editorial callouts. Optionally includes an author attribution line, an avatar image, and a title.

## When to use

Use `Quote` to highlight a significant statement from an article, case study, or customer testimonial. For inline citations use a standard `<blockquote>` element.

## Accessibility (WCAG 2.2)

- Content is rendered inside `<blockquote cite="…">` when a source URL is known
- Author attribution uses `<cite>` for semantic meaning
- Avatar images have meaningful `alt` text (author name) or `alt=""` when purely decorative

## ✅ Do

- Attribute every quote to a real author or source
- Keep quote text under ~200 characters for readability at display sizes

## ❌ Don't

- Don't use `Quote` for long body passages — use standard prose or a callout box
- Don't omit author attribution in customer-testimonial contexts where trust is critical

## API notes

- `color` is forwarded to the embedded `Avatar` and subtitle `Headline` for coordinated colouring
- The `color` prop now applies to the sub-headline, avatar, and the Card's border colour and background tint — all visual elements respond to a single token

## Design Tokens

Key `--pk-*` tokens used by this component:

| Token              | Purpose                                                    |
| ------------------ | ---------------------------------------------------------- |
| `--pk-space-xs`    | Author position top padding                                |
| `--pk-space-lg`    | Headline padding, avatar right margin, quote bottom margin |
| `--pk-color-fg`    | Author name text color                                     |
| `--pk-color-muted` | Author position/role text color                            |
