## 2. Globale Stile einbinden

Importiere das prokodo UI CSS-Bundle in deinem Root-Layout (`app/layout.tsx`):

```ts title="app/layout.tsx"
import '@prokodo/ui/theme.css'
```

Diese Datei:
- Importiert alle CSS-Kustomeigenschaften (`--pk-*` Design-Tokens)
- Stellt baseline-Resets bereit, die mit Next.js App Router kompatibel sind
- Muss **einmalig** am Root eingebunden werden — kein wiederholter Import pro Komponente erforderlich

:::info CSS-Architektur
Komponenten-CSS wird pro Komponente ausgeliefert (z. B. `@prokodo/ui/button.css`) und sollte zusammen mit der jeweiligen Komponente importiert werden. Beispiele findest du im [Schnellstart](/de/docs/getting-started/quick-start).
:::

---

## 3. next.config konfigurieren

Für die meisten Setups ist keine besondere Konfiguration erforderlich. Wenn du `transpilePackages` verwendest, musst du `@prokodo/ui` **nicht** hinzufügen — es wird als vorcompiliertes ESM ausgeliefert.

---

## 4. Überprüfen

Erstelle eine schnelle Testseite:

```tsx title="app/test/page.tsx"
import { Button } from '@prokodo/ui/button'
import '@prokodo/ui/button.css'

export default function TestPage() {
  return <Button>Es funktioniert!</Button>
}
```

Öffne `http://localhost:3000/test` — du solltest einen gestylten Button sehen.

---

## Fehlerbehebung

**`Cannot find module '@prokodo/ui/…'`** — Stelle sicher, dass `node_modules/@prokodo/ui` existiert. Führe erneut den Paketmanager-Install aus.

**Stile werden nicht geladen** — Bestätige, dass `@prokodo/ui/theme.css` im Root-Layout importiert ist, nicht in einer Komponentendatei.

**Icon wird leer dargestellt** — In der Entwicklung werden Icons aus dem lokalen `node_modules`-Asset-Pfad geladen. In der Produktion von jsDelivr CDN. Weitere Informationen auf der [Icon](/de/docs/components/icon)-Seite.
