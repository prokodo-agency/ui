# RTE

`RTE` (Rich Text Editor) bietet eine WYSIWYG-Bearbeitung mit konfigurierbarer Toolbar. Unterstützt Markdown-Shortcuts, Inline-Formatierung, Listen, Links und optionalen Bild-Upload.

:::warning Beta
`RTE` ist aktuell in **Beta**. API-Details und Verhalten können sich in kommenden Releases noch ändern.
:::

---

## Übersicht

```tsx
import { RTE } from "@prokodo/ui/rte"
;<RTE
  name="content"
  value={editorValue}
  onChange={setEditorValue}
  label="Article content"
/>
```

## Props

| Prop           | Typ                                              | Standard | Pflicht | Beschreibung                                               |
| -------------- | ------------------------------------------------ | -------- | ------- | ---------------------------------------------------------- |
| `id`           | `string`                                         | —        | —       | HTML-id für den Editor.                                    |
| `label`        | `string`                                         | —        | —       | Beschriftungstext oberhalb des Editors.                    |
| `value`        | `string`                                         | —        | —       | Gesteuerter HTML-Content-Wert.                             |
| `defaultValue` | `string`                                         | —        | —       | Unkontrollierter HTML-Ausgangswert.                        |
| `placeholder`  | `string`                                         | —        | —       | Platzhaltertext.                                           |
| `disabled`     | `boolean`                                        | `false`  | —       | Editor deaktivieren.                                       |
| `required`     | `boolean`                                        | `false`  | —       | Feld als Pflichtfeld markieren.                            |
| `errorText`    | `string`                                         | —        | —       | Fehlermeldung unterhalb des Editors.                       |
| `helperText`   | `string`                                         | —        | —       | Hilfetext unterhalb des Editors.                           |
| `rteOptions`   | `RTEOptions`                                     | —        | —       | Quill-Editor-Optionen.                                     |
| `rteToolbar`   | `RTEFeature[]`                                   | —        | —       | Toolbar-Feature-Array-Override.                            |
| `onChange`     | `(html: string, meta: { text: string }) => void` | —        | —       | Wird bei Inhaltsänderung mit HTML und Klartext aufgerufen. |
| `className`    | `string`                                         | —        | —       | CSS-Klasse am Root-Element.                                |

## Design-Tokens

Passe `RTE` über CSS Custom Properties auf `:root` oder einem übergeordneten Element an.

| Token                 | Standard                  | Beschreibung              |
| --------------------- | ------------------------- | ------------------------- |
| `--pk-rte-bg`         | `var(--pk-color-surface)` | Editor-Hintergrund        |
| `--pk-rte-toolbar-bg` | `color-mix(…)`            | Toolbar-Hintergrundtönung |
| `--pk-rte-accent`     | `var(--pk-color-brand)`   | Aktives Toolbar-Element   |
| `--pk-rte-icon-color` | `var(--pk-color-fg)`      | Icon-Farbe                |
| `--pk-rte-code-bg`    | `color-mix(…)`            | Inline-Code-Hintergrund   |
| `--pk-rte-radius`     | `var(--pk-radius-md)`     | Editor-Eckenradius        |

## AIC-Hinweis

Verwende im Anwendungscode immer den Standard-Importpfad:

```ts
import { RTE } from "@prokodo/ui/rte"
```

Eine separate Auswahl von `/client` oder `/lazy` ist im Consumer-Code nicht erforderlich.

AIC-Komponenten unterstützen außerdem ein `priority`-Flag für kritische Above-the-fold-Elemente.
Am sichtbarsten ist das bei `Image` (`loading="eager"` + `fetchpriority="high"`).
