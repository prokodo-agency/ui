# Switch

`Switch` rendert einen barrierefreien Toggle-Schalter mit Label, Disabled-State und semantisch korrekter ARIA-Rolle.

---

## Übersicht

```tsx
import { Switch } from "@prokodo/ui/switch"
;<Switch
  name="notifications"
  label="Email notifications"
  checked={enabled}
  onChange={e => setEnabled(e.target.checked)}
/>
```

## Props

| Prop          | Typ                                                                | Standard | Pflicht | Beschreibung                                 |
| ------------- | ------------------------------------------------------------------ | -------- | ------- | -------------------------------------------- |
| `id`          | `string`                                                           | —        | —       | HTML-id für das Switch-Input.                |
| `label`       | `string`                                                           | —        | —       | Beschriftungstext.                           |
| `checked`     | `boolean`                                                          | —        | —       | Gesteuerter Zustand (aktiviert/deaktiviert). |
| `disabled`    | `boolean`                                                          | `false`  | —       | Switch deaktivieren.                         |
| `required`    | `boolean`                                                          | `false`  | —       | Feld als Pflichtfeld markieren.              |
| `hideLabel`   | `boolean`                                                          | `false`  | —       | Beschriftung visuell ausblenden.             |
| `icon`        | `string`                                                           | —        | —       | Icon-Name für den deaktivierten Zustand.     |
| `checkedIcon` | `string`                                                           | —        | —       | Icon-Name für den aktivierten Zustand.       |
| `color`       | `Variants`                                                         | —        | —       | Farbakzent-Variante.                         |
| `labelProps`  | `LabelProps`                                                       | —        | —       | Props für das Label-Element.                 |
| `helperText`  | `string`                                                           | —        | —       | Hilfetext unterhalb des Schalters.           |
| `errorText`   | `string`                                                           | —        | —       | Fehlermeldung.                               |
| `onChange`    | `(event: ChangeEvent<HTMLInputElement>, checked: boolean) => void` | —        | —       | Wird bei Zustandsänderung aufgerufen.        |
| `className`   | `string`                                                           | —        | —       | CSS-Klasse am Root-Element.                  |

## Design-Tokens

Passe `Switch` über CSS Custom Properties auf `:root` oder einem übergeordneten Element an.

| Token                          | Standard                | Beschreibung               |
| ------------------------------ | ----------------------- | -------------------------- |
| `--pk-switch-track-width`      | `44px`                  | Track-Breite               |
| `--pk-switch-track-height`     | `24px`                  | Track-Höhe                 |
| `--pk-switch-track-bg`         | `color-mix(…)`          | Nicht-aktive Track-Farbe   |
| `--pk-switch-track-checked-bg` | `var(--pk-color-brand)` | Aktive Track-Farbe         |
| `--pk-switch-thumb-size`       | `18px`                  | Thumb-Durchmesser          |
| `--pk-switch-label-fg`         | `var(--pk-color-fg)`    | Label-Textfarbe            |
| `--pk-switch-active`           | `var(--pk-color-brand)` | Aktiv- / Fokus-Akzentfarbe |
| `--pk-switch-timing`           | `180ms`                 | Animationsdauer            |

## AIC-Hinweis

Verwende im Anwendungscode immer den Standard-Importpfad:

```ts
import { Switch } from "@prokodo/ui/switch"
```

Eine separate Auswahl von `/client` oder `/lazy` ist im Consumer-Code nicht erforderlich.

AIC-Komponenten unterstützen außerdem ein `priority`-Flag für kritische Above-the-fold-Elemente.
Am sichtbarsten ist das bei `Image` (`loading="eager"` + `fetchpriority="high"`).
