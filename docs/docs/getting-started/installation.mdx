## 2. Add global styles

Import the prokodo UI CSS bundle in your root layout (`app/layout.tsx`):

```ts title="app/layout.tsx"
import '@prokodo/ui/theme.css'
```

This file:
- Imports all CSS custom properties (`--pk-*` design tokens)
- Provides baseline resets compatible with Next.js App Router
- Must be imported **once** at the root — no need to repeat it per component

:::info CSS architecture
Individual component CSS is shipped per component (e.g. `@prokodo/ui/button.css`) and should be imported alongside each component. See [Quick Start](/docs/getting-started/quick-start) for examples.
:::

---

## 3. Configure next.config

No special configuration is required for most setups. If you use `transpilePackages`, you do **not** need to add `@prokodo/ui` — it ships pre-compiled ESM.

---

## 4. Verify

Create a quick test page:

```tsx title="app/test/page.tsx"
import { Button } from '@prokodo/ui/button'
import '@prokodo/ui/button.css'

export default function TestPage() {
  return <Button>It works!</Button>
}
```

Open `http://localhost:3000/test` — you should see a styled button.

---

## Troubleshooting

**`Cannot find module '@prokodo/ui/…'`** — Ensure `node_modules/@prokodo/ui` exists. Run your package manager install again.

**Styles not loading** — Confirm `@prokodo/ui/theme.css` is imported in the root layout, not inside a component file.

**Icon renders as empty** — In development, icons load from the local `node_modules` asset path. In production, they load from jsDelivr CDN. See the [Icon](/docs/components/icon) page.
